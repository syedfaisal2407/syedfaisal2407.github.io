# Faisal Ahamed Syed — Security Portfolio

Static portfolio site. No frameworks, no build step, no dependencies. One HTML file.

## Deploy to GitHub Pages (2 minutes)

1. Create a repo named **`syedfaisal2407.github.io`** (use your actual GitHub username — this exact name gives you the root URL).
2. Upload `index.html` (and `resume.pdf` if you have one) to the repo root.
3. Repo **Settings → Pages → Source: Deploy from a branch → main → / (root) → Save**.
4. Site is live at `https://syedfaisal2407.github.io` within a minute or two.

## Before you publish — required edits

Search `index.html` for `syedfaisal2407` (5 occurrences) and replace with your real GitHub / TryHackMe usernames.

- Project repo links point to repos that **must exist**. A portfolio linking to 404s is worse than no portfolio. Create the repos (even with just a README + screenshots) before going live, or remove the links.
- Drop your résumé PDF into the repo root as `resume.pdf`, or delete the download button.
- The phone number was intentionally left off the public site (scraper bait). Email + LinkedIn is enough.

## Custom domain (optional)

Buy a domain, add it under Settings → Pages → Custom domain, set the DNS CNAME to `syedfaisal2407.github.io`. Not required — the github.io URL is perfectly credible for entry-level.
